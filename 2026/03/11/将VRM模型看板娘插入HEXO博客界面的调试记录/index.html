基于HEXO的VRM看板娘调试经验

VRM看板娘调试经历

我最近尝试把VRM模型放在Solitude主题的博客左下角，实现类似live2d的动态跟随效果。
这个过程痛苦又充满教训，所以决定记录下来，供自己和大家参考。
刚开始会出现下拉大片空白页，模型背对着你，头到处飞的情况。

1️⃣ 初始问题

最开始模型可以正常显示，但遇到几个核心问题：

  1. 手臂始终T字
  2. 眼球不动或消失
  3. 鼠标追踪不自然
  4. 手臂方向不对（举到头上）（赞美太阳.jpg）

起初尝试用 getNormalizedBoneNode("leftUpperArm") 旋转手臂，但每帧都会被 VRM 自动重置到标准T Pose，所以看起来完全无效。

2️⃣ 研究原因（帮帮我，g老师）

通过查阅 VRM官方文档 和 Three.js 的 VRMLoader：

  • NormalizedBone 每帧会被 VRM Humanoid 重置 → 不能用来直接设置自然手臂姿态
  • RawBone 不会被重置，但 VRM3 规范推荐使用 枚举 VRMHumanBoneName
  • 手臂方向错误是因为 Z轴正负控制手臂上下方向
  • 眼睛骨骼也必须用枚举，否则看起来像眼球消失

3️⃣ 最终解决方案

✅ 核心做法

  1. 使用 getNormalizedBoneNode(VRMHumanBoneName.*) 获取骨骼
  2. 旋转 UpperArm 和 LowerArm 骨骼来自然下垂
  3. 鼠标移动平滑追踪头部和眼球
  4. 添加呼吸和眨眼动作
  5. 用点击事件触发互动对话

4️⃣ 展示部分

手臂自然下垂（调整了好久一直在赞美太阳，我说你真的是太阳战士吗）

::: code

function applyArmPose(){
  const lArm=currentVrm.humanoid.getNormalizedBoneNode(VRMHumanBoneName.LeftUpperArm)
  const rArm=currentVrm.humanoid.getNormalizedBoneNode(VRMHumanBoneName.RightUpperArm)
  const lLower=currentVrm.humanoid.getNormalizedBoneNode(VRMHumanBoneName.LeftLowerArm)
  const rLower=currentVrm.humanoid.getNormalizedBoneNode(VRMHumanBoneName.RightLowerArm)

  if(lArm){ lArm.rotation.z=-1.35; lArm.rotation.x=0.08 }
  if(rArm){ rArm.rotation.z=1.35; rArm.rotation.x=0.08 }

  if(lLower){ lLower.rotation.z=0.05 }
  if(rLower){ rLower.rotation.z=-0.05 }
}

头部与眼球追踪（折叠）

::: code

function updateLook(){
  targetX+=(mouseX-targetX)*0.08
  targetY+=(mouseY-targetY)*0.08

  const neck=currentVrm.humanoid.getNormalizedBoneNode(VRMHumanBoneName.Neck)
  const head=currentVrm.humanoid.getNormalizedBoneNode(VRMHumanBoneName.Head)
  const eyeL=currentVrm.humanoid.getNormalizedBoneNode(VRMHumanBoneName.LeftEye)
  const eyeR=currentVrm.humanoid.getNormalizedBoneNode(VRMHumanBoneName.RightEye)

  if(neck){ neck.rotation.y=targetX*0.5; neck.rotation.x=targetY*0.25 }
  if(head){ head.rotation.y=targetX*0.35; head.rotation.x=targetY*0.35 }

  if(eyeL&&eyeR){
    eyeL.rotation.y=targetX*0.12; eyeR.rotation.y=targetX*0.12
    eyeL.rotation.x=targetY*0.08; eyeR.rotation.x=targetY*0.08
  }
}

点击触发对话（折叠）

::: code

window.addEventListener("click",()=>{
  if(!currentVrm) return
  currentVrm.expressionManager.setValue("happy",1)
  setTimeout(()=>{currentVrm.expressionManager.setValue("happy",0)},800)
  showDialog("怎么了？有什么能帮到你吗？",4000)
})

:::

进入页面显示欢迎语（折叠）

::: code

loader.load("./VRM/prdx.vrm",(gltf)=>{
  const vrm=gltf.userData.vrm
  currentVrm=vrm
  showDialog("欢迎来到提斯沙盘~")
})

:::

5️⃣ 经验总结

VRM骨骼必须用枚举，否则修改无效

NormalizedBone每帧被Humanoid重置，手臂必须用 RawBone 或调试过的 Z/X 角度

眼球追踪角度要小，否则会失真

鼠标平滑追踪可以用简单插值 target += (mouse-target)*0.08

点击互动、欢迎语等 UI 可以用简单 DOM 悬浮气泡实现
